# Auditor 1 Report — Structural & Security Focus

> Date: 2026-06-01 · Focus: Input validation, security, error handling

## Findings

### FINDING-1A: SSE error events silently discarded (CONFIRMED)
**File**: `client/src/lib/api.ts:175-178`
The inner `catch {}` catches `throw new Error(data.error)` from server error events. This is a bug — error events are designed by the server (sse-responder.ts:7) but never reach the user. The fix should separate JSON parse errors from semantic error events.

### FINDING-1B: No fetch timeout (CONFIRMED)
**File**: `client/src/lib/api.ts:137`
No AbortController. On Vercel serverless with 30s max duration, the client fetch can hang for 30s+ with no feedback. This is a user experience bug — the loading indicator never stops.

### FINDING-1C: No history validation (CONFIRMED)
**File**: `server/src/routes/chat.ts:25-36`
The `message` field is validated but `history` is not. Malformed history items propagate to DeepSeek API. Low severity since only authenticated users can send, but still a missing validation boundary.

### FINDING-1D: Ghost assistant message (CONFIRMED)
**File**: `client/src/hooks/useChat.ts:34-40`
Empty assistant message created before API call. On immediate failure, the message persists. The store's `clear()` method is the only way to remove it currently. This is a UI integrity bug — the user sees a blank bubble that never populated.

### FINDING-1E: Anonymous rate limit bucket (CONFIRMED)
**File**: `server/src/routes/chat.ts:13`
All unauthenticated users share one "anonymous" rate limit bucket. This is a security concern — a single abusive IP can exhaust the shared bucket, blocking all unauthenticated users. The fix is to require authentication (return 401 if authUser is missing) rather than falling back to anonymous.

### FINDING-1F: No startup API key validation (CONFIRMED)
**File**: `server/src/lib/chat/deepseek-client.ts:3-6`
Empty key silently creates a broken client. Should validate at import time.
