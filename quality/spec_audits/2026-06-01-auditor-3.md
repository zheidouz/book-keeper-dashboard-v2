# Auditor 3 Report — PRD Compliance Focus

> Date: 2026-06-01 · Focus: Spec compliance against docs/prd-ai-chatbot.md

## Findings

### FINDING-3A: No rate limiting on anonymous users (CONFIRMED)
**PRD reference**: Section 4 — "Rate-limited per user (30 req/min) to control cost"
**File**: `server/src/routes/chat.ts:13`
The PRD specifies rate limiting per user. The implementation falls back to anonymous bucket when authUser is missing. This means unauthenticated users are not properly rate-limited per user — they share one bucket. This is a PRD compliance gap.

### FINDING-3B: SSE streaming works but error events not displayed (CONFIRMED)
**PRD reference**: Section 4 — SSE streaming response
**File**: `client/src/lib/api.ts:175-178`
The PRD specifies streaming response. The streaming itself works (tokens arrive). However, error events sent mid-stream are swallowed. The PRD's requirement for "Stream response back via SSE" is partially met — happy path works, error path doesn't.

### FINDING-3C: Message length capped, but history not validated (CONFIRMED)
**PRD reference**: Section 4 — "User messages capped at 2000 characters server-side"
**File**: `server/src/routes/chat.ts:35-36`
The 2000-char message cap is implemented (`message.trim().slice(0, 2000)`). However, the PRD does not specify history validation, so this is not a PRD gap per se. But best practice suggests validating history structure.

### FINDING-3D: Output token cap implemented (PASS)
**PRD reference**: Section 4 — "Output capped at 500 tokens"
**File**: `server/src/lib/chat/deepseek-client.ts:7,17`
`MAX_TOKENS` defaults to 500 and is configurable via `CHAT_MAX_TOKENS`. Pass.

### FINDING-3E: System prompt disclaimer present (PASS)
**PRD reference**: Section 3 — "AI-generated — verify with a qualified professional."
**File**: `server/src/lib/chat/prompt-builder.ts:47`
The disclaimer is included in the system prompt. Pass.

### FINDING-3F: No PII stored, ephemeral conversation (PASS)
**PRD reference**: Section 4 — "Chat history is ephemeral (Zustand in-memory only)"
**File**: `client/src/stores/chat-store.ts`
Messages stored only in Zustand (client memory). Page refresh clears history. Pass.

### FINDING-3G: Rate limiting implemented but per-instance (CONCERN)
**PRD reference**: Section 4 — "Rate-limited per user (30 req/min)"
**File**: `server/src/routes/chat.ts:9-13`
Rate limiting is implemented but is per-serverless-instance. The PRD doesn't specify the mechanism, so technically compliant. But on Vercel, the rate limit is not globally enforced.
