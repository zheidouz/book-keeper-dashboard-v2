# Integration Test Protocol: AI Bookkeeper Chatbot

> Generated by Quality Playbook v1.5.6 — Andrew Stellman
> Date: 2026-06-01 · Project: book-keeper-dashboard-v2

---

## Safety Constraints

- Do NOT call the real DeepSeek API — mock via `vi.mock()` or fetch interception
- Do NOT send real user messages to external APIs
- Do NOT use production Clerk credentials — test tokens only
- All tests must run in isolation (no shared state)

## Pre-flight Checks

1. `server/tsconfig.json` compiles (tsc --noEmit)
2. `client/tsconfig.json` compiles (tsc --noEmit)
3. All existing tests pass (`npm test` in client and server)

## Test Matrix

All commands run from project root using relative paths.

| Group | UC | Command | Pass Criteria |
|-------|----|---------|---------------|
| 1. SSE parser | UC-01 | `cd client && npx vitest run quality/test_functional.ts --reporter=verbose` | All tests pass (0 failures, 0 errors) |
| 2. Chat store | UC-01 | Include in group 1 | State mutations correct |
| 3. Prompt builder | UC-01 | `cd server && npx vitest run quality/test_functional.ts --reporter=verbose` | All tests pass |
| 4. Markdown rendering | UC-01 | Include in group 1 | XSS escape, bold, code, lists |
| 5. SSE responder | UC-01 | Include in group 3 | SSE format correct |
| 6. InputBar behavior | UC-02 | Include in group 1 | Disabled, trim, empty guard |

## Execution UX

1. Show plan as numbered table before running
2. Report one-line progress: `✓ Group 1: SSE parser`, `✗ Group 3: Prompt builder`
3. Show summary table with pass/fail counts
