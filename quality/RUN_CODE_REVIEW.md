## Pass 1 — Structural Review

Read the following files and spot anomalies without referring to requirements:

- `server/src/routes/chat.ts` — POST handler, rate limiting, SSE
- `server/src/lib/chat/deepseek-client.ts` — LLM streaming
- `server/src/lib/chat/prompt-builder.ts` — system prompt assembly
- `server/src/lib/chat/form-retriever.ts` — DB query
- `server/src/lib/chat/sse-responder.ts` — SSE write utility
- `client/src/lib/api.ts` — SSE parser (chatApi.sendStream section)
- `client/src/hooks/useChat.ts` — send + stream hook
- `client/src/stores/chat-store.ts` — Zustand state
- `client/src/components/chat/*.tsx` — UI components

**Mandatory scrutiny areas:**

1. **Input validation and boundary handling** — trust boundary at the HTTP route (req.body), at the SSE parser (incoming data lines), at the prompt builder (user message content)
2. **Resource lifecycle** — Response stream (res), fetch Response body reader, AbortController (or lack thereof)
3. **Concurrency and state management** — Zustand store mutation patterns, race between appendToken and setLoading
4. **Error handling** — catch blocks, error propagation, user-visible error messages
5. **Enumeration completeness** — None expected (no dispatch tables in chat module)

**Output**: Report findings with file:line, labeled BUG/CONCERN/QUESTION.

## Pass 3 — Cross-Requirement Consistency

Compare pairs of requirements that reference overlapping concerns:

- REQ-001 (SSE error visibility) × REQ-003 (ghost messages on failure) — Do both handle the mid-stream failure scenario consistently?
- REQ-002 (SSE timeout) × REQ-006 (retry on transient failures) — Does the timeout interact correctly with retries?
- REQ-004 (auth required) × REQ-005 (startup validation) — Do both enforcement points exist?

**Output**: Report contradictions or gaps where two requirements individually satisfied but together produce incorrect behavior.

## Bug Confirmation

For each VIOLATED requirement:
1. Write regression test in `quality/test_regression.ts` with `test.fails("BUG-NNN: ...", () => {...})`
2. Generate `quality/patches/BUG-NNN-regression-test.patch`
3. Optionally generate `quality/patches/BUG-NNN-fix.patch`
4. Append to BUGS.md

## Regression Test Skip Guards

All regression tests in `quality/test_regression.ts` must use `test.fails("BUG-NNN: ...", () => {...})` — Vitest's equivalent of xfail. The guard message must reference the bug ID.
