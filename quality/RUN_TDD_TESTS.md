## Overview

For every confirmed bug in `quality/BUGS.md`, run the red-green TDD cycle:

1. **Red phase**: Run regression test against unpatched code — MUST FAIL
2. **Green phase**: Apply fix patch, run same test — MUST PASS

## Execution Steps

### Per-Bug Cycle

```bash
# Red phase
npx vitest run quality/test_regression.ts --reporter=verbose -t "BUG-NNN" > quality/results/BUG-NNN.red.log 2>&1
echo "RED" > quality/results/BUG-NNN.red.log.tmp
cat quality/results/BUG-NNN.red.log >> quality/results/BUG-NNN.red.log.tmp
mv quality/results/BUG-NNN.red.log.tmp quality/results/BUG-NNN.red.log

# Apply fix
git apply quality/patches/BUG-NNN-fix.patch

# Green phase
npx vitest run quality/test_regression.ts --reporter=verbose -t "BUG-NNN" > quality/results/BUG-NNN.green.log 2>&1
echo "GREEN" > quality/results/BUG-NNN.green.log.tmp
cat quality/results/BUG-NNN.green.log >> quality/results/BUG-NNN.green.log.tmp
mv quality/results/BUG-NNN.green.log.tmp quality/results/BUG-NNN.green.log

# Revert fix
git apply -R quality/patches/BUG-NNN-fix.patch
```

All commands run from project root.
