# Requirements: AI Bookkeeper Chatbot

> Generated by Quality Playbook v1.5.6 — Andrew Stellman
> Date: 2026-06-01 · Project: book-keeper-dashboard-v2

## Overview

The AI Bookkeeper Chatbot is a floating chat assistant embedded in a Philippine accounting dashboard. Users ask natural language questions about BIR forms (tax form codes, deadlines, filing frequencies) and receive streaming AI responses from DeepSeek V4 Flash. The chatbot is accessible from every authenticated page via a floating action button.

## Use Cases

### UC-01: User asks a BIR form question
**Actor**: Bookkeeper
**Trigger**: Types question and presses Enter
**Expected**: Streaming response with form details (code, deadline, frequency)

### UC-02: User sends message while streaming
**Actor**: Any user
**Trigger**: Clicks Send while previous response is still loading
**Expected**: Input disabled, second click ignored

### UC-03: API call fails
**Actor**: Any user
**Trigger**: Network error or server error during chat
**Expected**: Error message visible, no ghost messages, loading stops

### UC-04: User clears chat history
**Actor**: Any user
**Trigger**: Clicks "New Chat" icon
**Expected**: All messages removed, empty state restored

## Requirements

### SSE Streaming

**REQ-001 (specific)**: SSE error events from the server must be visible to the user via `callbacks.onError()`. Error events must not be silently swallowed by the SSE parser's catch block.
- **Tier**: 3
- **File**: `client/src/lib/api.ts:175-178`
- **Conditions**: Server writes `data: {"error":"..."}` → client calls `onError` → error displayed in chat panel
- **Use cases**: UC-03
- **Bug**: BUG-002 (HIGH)

**REQ-002 (specific)**: The SSE `fetch()` call must use an AbortController with a configurable timeout (default: 30s). Timeout must trigger `onError` with "Request timed out".
- **Tier**: 3
- **File**: `client/src/lib/api.ts:137`
- **Conditions**: Server unresponsive for 30s → fetch aborted → `onError` fires → loading stops
- **Use cases**: UC-03
- **Bug**: BUG-003 (HIGH)

**REQ-003 (specific)**: If the API call fails before any tokens are received, the empty assistant message must be removed from the chat store. The user must see the error instead.
- **Tier**: 3
- **File**: `client/src/hooks/useChat.ts:34-40`
- **Conditions**: API throws → empty message removed → error shown
- **Use cases**: UC-03
- **Bug**: BUG-005 (MEDIUM)

### Input Validation

**REQ-004 (specific)**: The `/api/chat` endpoint must return 401 when `req.authUser` is null/undefined. Must not fall back to anonymous rate limiting.
- **Tier**: 3
- **File**: `server/src/routes/chat.ts:13`
- **Conditions**: No valid JWT → 401 response, no anonymous bucket
- **Bug**: BUG-002 (partial)

**REQ-005 (specific)**: Missing or empty `DEEPSEEK_API_KEY` env var must throw at module initialization time. Must not silently create a broken client.
- **Tier**: 3
- **File**: `server/src/lib/chat/deepseek-client.ts:3-6`
- **Conditions**: Env var unset → import throws → server fails at startup with clear message
- **Bug**: BUG-006 (MEDIUM)

**REQ-006 (specific)**: Transient failures from DeepSeek API (network errors, 5xx) must trigger up to 2 retries with exponential backoff (1s, 2s).
- **Tier**: 4 (inferred from implementation plan)
- **File**: `server/src/lib/chat/deepseek-client.ts:15-40`
- **Conditions**: First call fails → retry after 1s → retry after 2s → propagate if all fail
- **Bug**: BUG-004 (MEDIUM)

**REQ-007 (specific)**: The server must validate each item in the `history` array for `role` (user/assistant) and `content` (string, non-empty). Invalid items must be dropped or return 400.
- **Tier**: 3
- **File**: `server/src/routes/chat.ts:36`
- **Conditions**: Malformed history → 400 or filtered
- **Bug**: BUG-001 (LOW)

### Error Handling

**REQ-008 (specific)**: If the SSE ReadableStream `reader.read()` throws mid-stream (connection dropped), `callbacks.onError` must be called with the error and loading must stop.
- **Tier**: 3
- **File**: `client/src/lib/api.ts:137-194`
- **Conditions**: Connection drops → `onError` fires → loading stops → partial message preserved
- **Bug**: covered by BUG-002/BUG-003
